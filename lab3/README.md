# Лабораторная работа №3: Синхронизация потоков

## Описание
Программа демонстрирует синхронизацию потоков с помощью событий Windows.

**Суть:** Несколько потоков-маркеров пытаются заполнить массив своими номерами. Когда маркер не может поставить число (ячейка занята), он сообщает об этом главному потоку и засыпает. Главный поток периодически выводит массив и завершает один из маркеров.

## Алгоритм

### Главный поток
1. Создает массив, заполняет нулями
2. Создает события:
   - `startEvent` (ручной сброс) — запуск всех маркеров
   - `continueEvent` (автосброс) — продолжить работу
   - `terminateEvent` (автосброс) — завершить маркер
   - `stopEvents[i]` (автосброс) — маркер сообщает, что заблокирован
3. Запускает маркеры
4. Ждет, пока все маркеры не заблокируются
5. Выводит массив
6. Запрашивает номер маркера для завершения
7. Отправляет сигнал завершения, ждет завершения
8. Выводит массив после очистки
9. Дает сигнал оставшимся маркерам продолжить
10. Повторяет, пока все маркеры не завершены

### Поток маркера
1. Инициализирует генератор случайных чисел (srand(markerId))
2. В цикле:
   - Генерирует индекс: `rand() % размер`
   - Если ячейка = 0:
     - Sleep(5)
     - Записывает свой ID
     - Sleep(5)
     - Запоминает индекс
   - Если ячейка занята:
     - Выводит: `Marker X | Marked: Y | Blocked: Z`
     - Сигналит главному потоку через `stopEvent`
     - Ждет сигнала (продолжить или завершить)
3. При получении сигнала завершения:
   - Очищает все свои ячейки (ставит 0)
   - Завершается
